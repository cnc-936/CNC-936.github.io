Spark Local与Standalone模式部署

一、实验目的

  1. 理解Spark Local模式与Standalone模式的核心工作原理、适用场景差异。
  2. 掌握Spark Standalone独立集群的部署流程，实现1主2从集群的搭建与启动。
  3. 会在两种模式下提交Spark任务，对比任务运行的资源调度方式。
  4. 熟悉Spark Web UI的使用，能通过UI查看集群状态和任务运行详情。
  5. 掌握Standalone集群的启停命令，排查集群启动的常见问题。

二、实验环境

2.1 基础环境

已完成JDK 1.8、Scala 2.12.15、Spark 3.5.0基础环境配置，且所有节点环境一致。

2.2 集群节点规划（CentOS 7）

节点角色 节点IP地址 节点名称 配置说明
Master 192.168.10.101 master 集群管理节点
Worker1 192.168.10.102 worker1 工作节点，2核2G
Worker2 192.168.10.103 worker2 工作节点，2核2G

2.3 前置条件

所有节点之间实现SSH免密登录，关闭防火墙和SELinux，保证节点间网络互通。

三、实验步骤

1. Spark Local模式部署与任务提交

Local模式是Spark的单机本地运行模式，无集群依赖，所有任务均在本地JVM中执行，仅适用于开发、调试和小数据量测试。

1.1 启动指定核心数的Local模式

# 方式1：使用1个核心运行
spark-shell --master local
# 方式2：使用4个核心运行（推荐，充分利用本地资源）
spark-shell --master local[4]
# 方式3：使用本地所有可用核心运行
spark-shell --master local[*]
1.2 提交 WordCount 单词计数任务
scala
# 1. 在本地创建测试文本文件
// 退出Spark Shell，执行创建文件命令
:quit
echo "hello spark hello hadoop spark local" > /usr/local/test.txt
# 2. 重新启动Local模式Spark Shell
spark-shell --master local[4]
# 3. 执行WordCount任务
val textFile = sc.textFile("file:///usr/local/test.txt")  // 读取本地文件
val wordCount = textFile.flatMap(_.split(" "))          // 按空格切分单词
                        .map((_,1))                    // 每个单词标记为1
                        .reduceByKey(_+_)              // 按单词聚合计数
wordCount.collect().foreach(println)                   // 收集并输出结果
# 成功输出如下结果即为任务执行完成
(hello,2)
(spark,2)
(hadoop,1)
(local,1)
1.3 Local 模式核心特点
部署零成本，无需配置集群，开箱即用；
所有任务在本地执行，无网络传输开销，调试方便；
无法利用分布式资源，仅适用于开发测试，不支持生产环境。
2. Spark Standalone 模式集群部署
Standalone 模式是 Spark 自带的独立集群模式，由 Spark 自身提供集群管理和资源调度，无需依赖第三方资源管理器（如 YARN、Mesos），适用于中小型 Spark 集群部署。
2.1 配置 Master 主节点
修改 workers 配置文件：指定集群工作节点
# 进入Spark配置目录
cd $SPARK_HOME/conf
# 复制配置模板
cp workers.template workers
# 编辑workers文件，添加两个Worker节点IP
vim workers
# 删除原有内容，添加
192.168.10.102
192.168.10.103
修改 spark-env.sh 配置文件：指定 Master 地址和节点资源
vim spark-env.sh
# 在文件中添加以下配置
export SPARK_MASTER_HOST=192.168.10.101  # Master节点IP
export SPARK_MASTER_PORT=7077            # Spark集群默认端口
export SPARK_WORKER_CORES=2              # 每个Worker节点可用核心数
export SPARK_WORKER_MEMORY=2g            # 每个Worker节点可用内存
export SPARK_WORKER_DIR=/usr/local/spark/worker  # Worker节点工作目录
2.2 分发 Spark 配置到所有 Worker 节点
将 Master 节点配置好的 Spark 目录，分发到 Worker1 和 Worker2 节点，保证所有节点配置一致：
# 分发Spark目录到worker1
scp -r /usr/local/spark root@192.168.10.102:/usr/local/
# 分发Spark目录到worker2
scp -r /usr/local/spark root@192.168.10.103:/usr/local/
# 验证分发结果：在Worker节点查看Spark目录是否存在
ssh root@192.168.10.102 "ls /usr/local/spark"
2.3 启动 Standalone 集群
仅在 Master 节点执行启停命令，无需在 Worker 节点单独操作：
# 启动集群
start-all.sh
# 查看集群进程（Master节点）
jps
# 成功显示Master进程即为启动成功
Master
# 查看Worker节点进程（Worker1/Worker2）
jps
# 成功显示Worker进程即为启动成功
Worker
2.4 查看集群 Web UI 状态
打开浏览器，访问 Master 节点的 Spark Web UI 地址，查看集群状态：
plaintext
http://192.168.10.101:8080
页面显示2 个 Worker 节点在线，与配置一致；
可查看集群总核心数、总内存、任务运行记录等信息；
这是监控 Standalone 集群的核心入口。
3. Standalone 集群提交任务
3.1 启动连接集群的 Spark Shell
# 在Master节点执行，连接Standalone集群
spark-shell --master spark://192.168.10.101:7077
3.2 提交分布式 WordCount 任务
# 1. 将测试文件分发到所有节点的相同路径
scp /usr/local/test.txt root@192.168.10.102:/usr/local/
scp /usr/local/test.txt root@192.168.10.103:/usr/local/
# 2. 执行分布式WordCount任务
val textFile = sc.textFile("file:///usr/local/test.txt")
val wordCount = textFile.flatMap(_.split(" ")).map((_,1)).reduceByKey(_+_)
wordCount.collect().foreach(println)
# 结果与Local模式一致，任务由集群2个Worker节点分布式执行
3.3 停止 Standalone 集群
# 在Master节点执行停止命令
stop-all.sh
# 验证进程：jps查看无Master/Worker进程即为停止成功
四、实验常见问题与解决
1.Worker 节点无法连接 Master：检查节点间 SSH 免密登录是否配置成功，防火墙是否关闭，Spark 配置文件中 IP 是否拼写正确。
2.Web UI 无法访问：检查 Master 节点 8080 端口是否开放，spark-env.sh中SPARK_MASTER_HOST是否为节点公网 / 内网 IP（勿用 127.0.0.1）。
3.集群启动后 Worker 节点离线：检查所有节点 Spark 版本、配置文件、环境变量完全一致，Worker 节点工作目录有读写权限。
五、实验总结
本次实验完成了 Spark Local 模式和 Standalone 模式的全流程部署，明确了两种模式的核心差异：Local 模式适用于开发调试，Standalone 模式实现了分布式集群运行。掌握了 Standalone 集群 1 主 2 从的配置、分发、启停流程，会通过 Spark Web UI 监控集群状态，能在两种模式下提交 WordCount 任务并对比运行方式。同时解决了集群部署中的 SSH 免密、配置分发、节点连接等常见问题，理解了 Spark 独立集群的资源调度逻辑，为后续高可用集群和 YARN 模式部署打下基础。
