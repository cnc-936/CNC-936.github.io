Spark基础环境配置教程

一、实验目的

  1. 掌握Spark运行核心依赖环境（JDK、Scala）的安装与环境变量配置方法。
  2. 完成Spark 3.5.0单机版的解压、配置，实现基础环境的可用性验证。
  3. 熟悉Spark Shell交互式工具的使用，为后续集群模式部署和应用开发铺垫基础。
  4. 理解Spark与JDK、Scala的版本兼容原则，规避基础环境配置的常见问题。

二、实验环境

软件/环境 版本 运行系统 说明
JDK 1.8.0_301 CentOS 7 / Win10 Spark官方推荐基础JDK版本
Scala 2.12.15 CentOS 7 / Win10 与Spark 3.x系列完美兼容
Spark 3.5.0-bin-hadoop3 CentOS 7 / Win10 适配Hadoop 3.x的稳定版
环境变量配置 系统级/profile CentOS 7 / Win10 全局生效，所有用户可访问

三、实验步骤

1. JDK环境安装与配置

1.1 解压安装包

将下载的JDK压缩包解压至指定目录（以CentOS 7为例）：

# 解压到/usr/local/目录
tar -zxvf jdk-8u301-linux-x64.tar.gz -C /usr/local/
# 重命名文件夹（方便后续配置）
mv /usr/local/jdk1.8.0_301 /usr/local/jdk1.8
1.2 配置系统环境变量
# 编辑系统环境变量文件
vim /etc/profile
# 在文件末尾添加以下配置
export JAVA_HOME=/usr/local/jdk1.8
export JRE_HOME=$JAVA_HOME/jre
export CLASSPATH=.:$JAVA_HOME/lib:$JRE_HOME/lib
export PATH=$PATH:$JAVA_HOME/bin
# 使配置立即生效
source /etc/profile
1.3 验证 JDK 安装
# 执行命令查看版本
java -version
# 成功输出如下信息即为配置完成
java version "1.8.0_301"
Java(TM) SE Runtime Environment (build 1.8.0_301-b09)
Java HotSpot(TM) 64-Bit Server VM (build 25.301-b09, mixed mode)
2. Scala 环境安装与配置
2.1 解压安装包
# 解压Scala压缩包到/usr/local/目录
tar -zxvf scala-2.12.15.tgz -C /usr/local/
# 重命名文件夹
mv /usr/local/scala-2.12.15 /usr/local/scala
2.2 配置系统环境变量
# 编辑/etc/profile文件
vim /etc/profile
# 末尾添加以下配置
export SCALA_HOME=/usr/local/scala
export PATH=$PATH:$SCALA_HOME/bin
# 使配置生效
source /etc/profile
2.3 验证 Scala 安装
# 执行命令查看版本
scala -version
# 成功输出如下信息即为配置完成
Scala code runner version 2.12.15 -- Copyright 2002-2021, LAMP/EPFL
3. Spark 单机版安装与配置
3.1 解压 Spark 安装包
# 解压Spark包到/usr/local/目录
tar -zxvf spark-3.5.0-bin-hadoop3.tgz -C /usr/local/
# 重命名文件夹
mv /usr/local/spark-3.5.0-bin-hadoop3 /usr/local/spark
3.2 配置 Spark 环境变量
# 编辑/etc/profile文件
vim /etc/profile
# 末尾添加以下配置
export SPARK_HOME=/usr/local/spark
export PATH=$PATH:$SPARK_HOME/bin:$SPARK_HOME/sbin
# 使配置生效
source /etc/profile
3.3 修改 Spark 核心配置文件
# 进入Spark配置目录
cd $SPARK_HOME/conf
# 复制配置模板为正式配置文件
cp spark-env.sh.template spark-env.sh
# 编辑配置文件，指定JDK路径
vim spark-env.sh
# 在文件开头添加
export JAVA_HOME=/usr/local/jdk1.8
4. Spark 环境可用性验证
4.1 启动 Spark Shell 交互式工具
# 直接执行命令启动
spark-shell
# 成功启动后进入Scala交互界面，显示Spark版本和Web UI地址
4.2 执行简单 RDD 操作验证
# 创建一个简单的RDD并执行计算
val rdd = sc.parallelize(1 to 10)  // 创建1-10的分布式数据集
val sum = rdd.sum()                // 计算数据集总和
println("RDD求和结果：" + sum)     // 输出结果
# 成功输出「RDD求和结果：55.0」即为环境完全可用
4.3 退出 Spark Shell
:quit  // 执行退出命令
四、实验常见问题与解决
1.环境变量配置后验证失败：检查/etc/profile文件中路径是否正确，是否执行source /etc/profile生效，重启终端重试。
2.Spark Shell 启动提示 Java 未找到：确认spark-env.sh中JAVA_HOME路径与实际 JDK 安装路径一致，无拼写错误。
3.Scala 版本不兼容：Spark 3.x 仅支持 Scala 2.12.x 系列，避免使用 2.11.x 或 2.13.x 版本。
五、实验总结
本次实验完成了 Spark 基础运行环境的全流程配置，掌握了 JDK、Scala、Spark 的逐层依赖配置逻辑，明确了版本兼容的核心原则。通过 Spark Shell 的 RDD 简单操作，验证了基础环境的可用性，解决了环境配置中的常见路径、环境变量生效问题，为后续 Spark Local 模式、Standalone 集群模式的部署奠定了坚实的基础。
